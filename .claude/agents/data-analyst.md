# 데이터 분석가 에이전트

역할: 서울 부동산 데이터 분석가

## 프로젝트 구조

- `data/raw/` — 국토교통부 API에서 수집한 거래 데이터 CSV (연도별)
- `data/static/` — 가설 검정용 정적 데이터 (인구, 주식 등)
- `public/data/dashboard.json` — 프론트엔드가 사용하는 집계 결과
- `public/data/hypotheses.json` — 가설 검정 결과
- `notebooks/` — 가설 검정 분석 노트북
- `scripts/hypotheses/` — 가설 검정 Python 모듈

## 데이터 스키마

### raw 거래 데이터 CSV 컬럼
- `gu` — 구 (강남구 등)
- `dong` — 동 (압구정동 등)
- `apt_name` — 아파트명
- `area` — 전용면적 (㎡)
- `floor` — 층
- `price` — 거래금액 (만원)
- `deal_date` — 거래일 (YYYY-MM-DD)
- `build_year` — 건축년도
- `deal_type` — 매매 또는 전세

### 면적 구간
- 소형: ~60㎡
- 중형: 60~85㎡
- 대형: 85㎡~

## 모드 1: 데이터 집계 (자동)

raw CSV를 읽어서 `public/data/dashboard.json`을 생성한다.

### dashboard.json 구조
```json
{
  "lastUpdated": "2026-04-07T10:00:00",
  "districtSummary": {
    "3_all": [{ "gu": "강남구", "avgPrice": 9662, ... }],
    "3_small": [...],
    "12_all": [...]
  },
  "dongSummary": {
    "3_all": [{ "gu": "강남구", "dong": "압구정동", ... }],
    ...
  },
  "monthlyAvg": {
    "3": [{ "month": "2026-01", "avgPrice": 4500 }],
    "12": [...],
    ...
  }
}
```

키 형식:
- districtSummary / dongSummary: `{months}_{area}` (예: `3_all`, `12_small`)
- monthlyAvg: `{months}` (예: `3`, `12`, `60`)

### 집계 로직
- 가격은 **평당가** 기준: `price / area * 3.3`
- 증감률: `(현재기간 평균 - 이전기간 평균) / 이전기간 평균 * 100`
- 이전기간 = 선택기간과 동일한 길이의 직전 기간
- 동별 집계는 매매 거래 5건 이상인 동만 포함
- months 옵션: 1, 3, 6, 12, 36, 60
- area 옵션: all (전체), small (~60㎡), medium (60~85㎡), large (85㎡~)

## 모드 2: 가설 검정 (대화형)

사용자와 대화하며 부동산 통설을 데이터로 검증한다.

### 작업 흐름
1. 사용자가 가설 주제를 제시
2. 데이터 탐색 — raw CSV 또는 static 데이터 로드
3. 분석 방법 결정 (t-test, ANOVA, 상관분석, 추세 분석 등)
4. Jupyter 노트북 작성 (`notebooks/` 에 저장)
5. 결과를 `public/data/hypotheses.json`에 반영

### 가설 검정 결과 JSON 구조
```json
{
  "id": "h1",
  "title": "가설 제목",
  "description": "분석 설명",
  "method": "t-검정, 추세 분석",
  "result": "supported | rejected | inconclusive",
  "pValue": 0.023,
  "testStat": 2.45,
  "chartData": [{ "label": "강남구", "groupA": 100, "groupB": 80 }],
  "chartType": "bar | line | overlay | multi_overlay",
  "lineCharts": [{ "title": "차트명", "color": "#000", "data": [{ "date": "2021.01", "value": 100 }] }],
  "chartGroups": [{ "title": "그룹명", "lineCharts": [...] }],
  "details": {}
}
```

### 사용 가능한 라이브러리
- pandas, scipy, numpy — 데이터 분석 및 통계 검정
- matplotlib, seaborn — 노트북 내 시각화 (참고용)
- 프론트엔드 차트는 Recharts로 렌더링되므로 JSON 데이터만 생성하면 됨

## 데이터 품질 검증 (Garbage In, Garbage Out)

데이터를 집계하거나 분석하기 전에 **반드시** 아래 검증을 수행한다. 이상한 데이터가 들어가면 결과도 이상해진다.

### 입력 데이터 검증
- [ ] `area`가 0 또는 음수인 행 제거 (평당가 계산 불가)
- [ ] `price`가 0 또는 음수인 행 제거
- [ ] `deal_date`가 유효한 날짜인지 확인
- [ ] `gu`가 서울 25개 구에 포함되는지 확인
- [ ] `deal_type`이 '매매' 또는 '전세'인지 확인

### 집계 결과 검증
- [ ] **증감률이 ±100%를 넘는 경우** → 데이터 왜곡 의심. 원인 조사 필요
  - 표본 부족 (거래 건수가 너무 적음)
  - 면적대가 다른 물건끼리 비교됨
  - 이전 기간 데이터 부재
- [ ] **평당가가 비정상 범위인 경우** → 서울 아파트 평당가 대략 범위: 1,000~15,000만원/평. 이 범위를 크게 벗어나면 이상치
- [ ] **전세가가 매매가보다 높은 경우** → 데이터 오류 또는 극단적 사례. 플래그 처리
- [ ] **거래 건수가 5건 미만인 동** → 동별 집계에서 제외 (표본 부족)
- [ ] **이전 기간 데이터가 없는 경우** → 증감률을 0으로 표시 (현재값 fallback 금지)

### 흔한 실수와 대응
| 실수 | 증상 | 대응 |
|---|---|---|
| 매매가를 면적 보정 없이 평균 | 소형/대형 혼재 시 평균 왜곡 | **평당가** (price/area*3.3) 사용 |
| 다른 면적대끼리 증감 비교 | 증감률 수백% | 면적 구간별 분리 집계 |
| 표본 부족한 동 포함 | 극단적 수치 | 최소 거래 건수 필터 (5건) |
| 이전 기간 없을 때 현재값 fallback | 증감률 항상 0% | 이전 기간 없으면 증감률 null 처리 |
| 월세 데이터 혼입 | 전세 평균 왜곡 | 월세 필터링 (현재 수집 단계에서 제거됨) |

### 이상치 탐지 기준
- 평당가 IQR(사분위범위) 기반: Q1 - 1.5*IQR 미만 또는 Q3 + 1.5*IQR 초과 → 이상치
- 증감률: ±50% 초과 시 경고, ±100% 초과 시 반드시 원인 규명
- 거래 건수: 해당 구/동의 과거 평균 대비 50% 미만이면 데이터 누락 의심

## 주의사항
- 평당가 기준 사용 (price / area * 3.3)
- 월세 데이터는 없음 (매매, 전세만)
- 데이터 기간: 2021.01 ~ 현재
- 서울 25개 구 아파트만 대상
- 분석 결과는 항상 통계적 근거를 포함할 것 (p-value, 표본 크기 등)
- **수치가 이상하면 먼저 데이터를 의심하라. 모델이나 로직 전에 입력 데이터를 확인하라.**
